## 🎨 示例 3：纯文本格式风格描述

### 场景：用户手动输入的风格描述

```
使用深色背景，现代简约风格，蓝色和橙色作为强调色。整体设计要专业、科技感强，适合技术类演示。字体使用无衬线字体，确保清晰易读。视觉元素要简洁，避免过度装饰。
```

**注意**：这种格式会被追加到 `extra_requirements` 中，作为额外要求传递给图片生成模型。

## 📊 字段详细说明

### 1. `design_aesthetic`（设计美学）

**作用**：描述整体设计理念和视觉风格

**内容特点**：
- 包含角色设定（"你是架构师..."）
- 包含核心指令（5 条设计原则）
- 包含具体的风格描述（根据内容主题生成）
- 包含视觉隐喻的描述

**示例片段**：
```
"未来科技感"与"学术严谨性"的结合。设计语言冷峻而理性，强调数据可视化和信息架构的美感。视觉隐喻围绕"神经网络"、"算法流程"和"智能进化"展开。
```

---

### 2. `background_color`（背景色）

**作用**：指定背景颜色和设计说明

**格式**：`"颜色名称 (英文名), #十六进制代码。设计说明。"`

**示例**：
```
"深空灰 (Deep Space Grey), #1A1A1A。这为高亮数据提供了极佳的对比度，减少屏幕眩光，适合深度阅读。背景可以带有极淡的网格纹理，暗示数据结构和算法逻辑。"
```

---

### 3. `primary_font`（标题字体）

**作用**：指定标题使用的字体和设计说明

**格式**：`"字体名称。使用说明和设计考量。"`

**示例**：
```
"Helvetica Now Display (Bold/Black)。用于极具冲击力的标题，字间距紧凑，传达权威感和科技感。标题应该占据视觉重心，成为页面的核心锚点。"
```

---

### 4. `secondary_font`（正文字体）

**作用**：指定正文使用的字体和设计说明

**格式**：`"字体名称。使用说明和设计考量。"`

**示例**：
```
"JetBrains Mono。一种为代码和数据设计的等宽字体，用于正文、标注和数据点。这种字体强化"计算"和"工程"的语境，特别适合展示技术细节、算法流程和数据可视化。"
```

---

### 5. `primary_text_color`（主要文字颜色）

**作用**：指定主要文字的颜色

**格式**：`"颜色名称 (英文名), #十六进制代码。设计说明。"`

**示例**：
```
"雾白 (Mist White), #E0E0E0。在深色背景上形成强烈对比，确保文字清晰可读。这种颜色既不会过于刺眼，又能保持足够的可见度。"
```

---

### 6. `accent_color`（强调色）

**作用**：指定用于高光、图表和关键元素的强调色

**格式**：`"颜色名称 (英文名), #十六进制代码 (代表含义) 和 颜色名称 (英文名), #十六进制代码 (代表含义)。"`

**示例**：
```
"信号橙 (Signal Orange), #FF5722 (代表重要突破/警示) 和 荧光青 (Cyber Cyan), #00BCD4 (代表技术进步/效率)。橙色用于标记关键历史节点和重要突破，青色用于表示技术演进和效率提升。"
```

---

### 7. `visual_elements`（视觉元素）

**作用**：详细描述视觉元素的风格和要求

**内容特点**：
- 描述背景纹理
- 描述几何体和图形元素
- 描述图表风格
- 描述视觉隐喻
- 描述禁止使用的元素

**示例**：
```
"细如发丝的白色网格线作为背景纹理，暗示数据结构和算法逻辑。使用半透明的几何体表示"神经网络节点"和"数据流"。图表应采用极简的线图或热力图风格，展示技术演进的时间线。可以使用渐变的光流效果表示"智能进化"的过程。避免任何装饰性的插画，一切元素必须服务于数据表达和技术展示。视觉元素应该具有"蓝图"和"工程图纸"的精密感，同时保持现代科技的美学。"
```

## 🔄 风格描述在系统中的流转

### 步骤 1：生成（步骤 2：生成大纲时）

**AI 生成的 JSON 格式**：
```json
{
  "style_instructions": {
    "design_aesthetic": "...",
    "background_color": "...",
    "primary_font": "...",
    "secondary_font": "...",
    "primary_text_color": "...",
    "accent_color": "...",
    "visual_elements": "..."
  },
  "outline": [...]
}
```

**保存到数据库**：
```python
project.template_style = json.dumps(style_instructions, ensure_ascii=False)
# 保存为 JSON 字符串
```

---

### 步骤 2：应用（步骤 3：生成页面描述时）

**从数据库读取**：
```python
style_instructions = json.loads(project.template_style)
```

**格式化到提示词**：
```
<style_instructions>
设计美学：你是架构师（The Architect）...未来科技感与学术严谨性的结合...
背景色：深空灰 (Deep Space Grey), #1A1A1A...
标题字体：Helvetica Now Display (Bold/Black)...
正文字体：JetBrains Mono...
主要文字颜色：雾白 (Mist White), #E0E0E0
强调色：信号橙 (Signal Orange), #FF5722 和 荧光青 (Cyber Cyan), #00BCD4
视觉元素：细如发丝的白色网格线作为背景纹理...
</style_instructions>
```

---

### 步骤 3：应用（步骤 4：生成页面图片时）

**无模板图模式**：
```
<style_instructions>
设计美学：你是架构师（The Architect）...未来科技感与学术严谨性的结合...
背景色：深空灰 (Deep Space Grey), #1A1A1A...
标题字体：Helvetica Now Display (Bold/Black)...
正文字体：JetBrains Mono...
主要文字颜色：雾白 (Mist White), #E0E0E0
强调色：信号橙 (Signal Orange), #FF5722 和 荧光青 (Cyber Cyan), #00BCD4
视觉元素风格：细如发丝的白色网格线作为背景纹理...
</style_instructions>
```

**有模板图模式**：
- ❌ 不插入 `<style_instructions>` 标签
- ✅ 模板图片是主要参考

## 💡 关键要点

1. **JSON 格式更精确**：
   - 结构化数据，每个字段都有明确的含义
   - 可以直接解析为字典对象
   - 在无模板图模式下，会插入到 `<style_instructions>` 标签中

2. **纯文本格式更灵活**：
   - 用户可以用自然语言描述
   - 会被追加到 `extra_requirements` 中
   - 作为补充说明传递给图片生成模型

3. **AI 自动生成**：
   - 会根据内容主题智能生成风格描述
   - 避免通用的"极简主义"或"商务风格"等泛化描述
   - 创造符合内容主题的独特视觉隐喻

4. **字段完整性**：
   - 所有 7 个字段都应该包含
   - 每个字段都应该有详细的描述，不仅仅是颜色代码或字体名称
   - 设计说明很重要，帮助 AI 理解设计意图
