# `design_aesthetic` 字段详细分析

## 📋 概述

本文档详细分析 `design_aesthetic` 字段中哪些部分是**写死的**，哪些是**根据主题生成的**。

## 🔍 代码中的提示词要求

在 `prompts.py::get_outline_generation_prompt()` 中，AI 被要求：

```
在生成风格指令时，请将上述架构师角色设定和核心指令融入到风格描述中，特别是design_aesthetic字段应体现这些设计原则。

**风格指令说明：**
- design_aesthetic: 根据具体内容和受众，使用独特且有创意的美学风格描述，避免通用的"极简主义"或"商务风格"等泛化描述。参考上述示例，创造符合内容主题的视觉隐喻。
```

**关键要求**：
- ✅ **必须包含**：角色设定和核心指令（写死的部分）
- ❌ **必须生成**：根据具体内容和受众，创造符合内容主题的视觉隐喻（动态部分）

## 🎯 总结

### 写死的部分（约 40-50%）

1. **角色设定**：`"你是架构师（The Architect）..."`
2. **核心指令**：5 条固定的设计原则
3. **连接词**：`"在此基础上，"`

**作用**：提供统一的设计基础和原则，确保所有 PPT 都遵循相同的设计哲学。

---

### 根据主题生成的部分（约 50-60%）

1. **风格融合描述**：根据主题选择合适的美学风格组合
2. **设计语言描述**：根据主题描述设计语言的特点
3. **视觉隐喻**：根据主题选择相关的视觉隐喻元素
4. **整体感觉**：根据主题描述整体感觉和氛围
5. **配色方案说明**：根据主题选择配色方案并说明原因

**作用**：根据具体内容主题，创造独特且符合主题的视觉风格，避免通用模板。

---

## 📝 关键要点

1. **固定框架 + 动态内容**：
   - 写死的部分提供统一的设计基础
   - 生成的部分根据主题创造独特风格

2. **避免泛化描述**：
   - AI 被要求避免使用"极简主义"、"商务风格"等泛化描述
   - 必须根据具体主题创造独特的视觉隐喻

3. **视觉隐喻的重要性**：
   - 视觉隐喻是生成部分的核心
   - 必须与内容主题紧密相关（如"神经网络"对应 AI 主题，"增长曲线"对应商务主题）

4. **整体一致性**：
   - 写死的部分确保所有 PPT 都遵循相同的设计原则
   - 生成的部分确保每个 PPT 都有独特的风格
